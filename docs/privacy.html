Privacy Policy

Last updated: July 6, 2026

This Privacy Policy describes how Listen on Repeat handles data.

Listen on Repeat is currently provided as a self-hosted/template project. It is intended for users who configure their own Discord Developer Application, Spotify Developer Application, GitHub Actions workflow, and secrets.

Summary

The self-hosted version of Listen on Repeat does not send your Spotify tokens, Discord tokens, or widget data to PikaChokeMe.

If you run your own copy, your data is processed by your own configured infrastructure, such as your GitHub Actions workflow and your own Discord and Spotify developer applications.

Data used by the project

Depending on how you configure it, Listen on Repeat may access Spotify data needed to display your widget, such as:

  • Track title
  • Artist name
  • Album name
  • Album artwork URL
  • Spotify track metadata returned by the Spotify API

This data is used to populate the Discord widget fields you configure.

Spotify authorization

Listen on Repeat is designed to use Spotify OAuth authorization. For the intended top-tracks widget, the relevant Spotify permission is read-only access to your top items.

The project does not need your Spotify password. You should never put your Spotify password, Spotify refresh token, Spotify access token, Discord bot token, Discord user token, or client secret into public repository files, public web pages, or client-side widget code.

Discord data

Listen on Repeat formats Spotify track data and pushes it to Discord as widget display data.

The current self-hosted template does not require access to your Discord messages, Discord servers, payment information, or private conversations.

Secrets and tokens

If you self-host this project, you are responsible for storing your own secrets safely.

Recommended secret storage includes GitHub Actions repository secrets or another trusted server-side secret store. Do not commit secrets to your repository.

Data collected by PikaChokeMe

For the current self-hosted/template version, PikaChokeMe does not operate a hosted backend for your account and does not collect your Spotify tokens, Discord tokens, or widget data.

If a hosted version of Listen on Repeat is offered in the future, it will need its own updated privacy policy describing what data is collected, how it is stored, and how users can disconnect or delete their data.

Third-party services

Your use of Listen on Repeat may involve third-party services, including Spotify, Discord, GitHub, and any hosting provider you choose.

Those services have their own privacy policies, terms, and data practices. You are responsible for reviewing and following them when configuring your own copy of this project.

Children's privacy

Listen on Repeat is not intended for children. Do not use this project if you are not old enough to use the relevant third-party services, including Spotify, Discord, and GitHub.

Changes to this policy

This Privacy Policy may be updated from time to time. Changes will be posted on this page with an updated date.

Contact

For questions about this project, open an issue in the Listen on Repeat GitHub repository: https://github.com/PikaChokeMe/recently-on-repeat/issues

Do not post Spotify tokens, Discord tokens, client secrets, refresh tokens, access tokens, or other private credentials in GitHub issues.
